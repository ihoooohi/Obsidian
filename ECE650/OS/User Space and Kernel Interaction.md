## 一、OS 的主要服务

OS 同时扮演两个角色：给用户提供接口（GUI / Shell / Batch），以及在底层提供服务。核心服务包括：程序执行、I/O 操作、文件系统、进程间通信（IPC）、资源分配、账务统计、保护与安全。

---

## 二、调用 OS 服务的两种方式

|方式|触发者|典型例子|
|---|---|---|
|**主动 (Active)**|程序自己发起|`fork()`, `read()`, `write()` 等系统调用|
|**被动 (Passive)**|外部异步事件触发|中断 (Interrupt)、异常 (Exception)、信号 (Signal)|

---

## 三、中断 (Interrupt) 机制

![[Pasted image 20260330101541.png]]

中断的分类是这节课的重点之一：**中断处理流程**（记住这个顺序很关键）：

1. 外设通过 **IRQ 线** 发出请求
2. CPU 完成当前指令周期后检测到 IRQ
3. **保存当前进程状态**（上下文切换）
4. 跳转到**中断向量表**，找到对应的 **ISR（中断服务程序）**
5. ISR 执行完毕，执行 `IRET`（x86）返回，恢复用户进程

---

## 四、信号 (Signal)

信号是 UNIX 在软件层面模拟中断的机制，让内核可以异步通知进程。**信号 vs 中断的关键对比**：

- 中断：硬件层面，CPU 直接响应 IRQ 线
- 信号：软件层面，由内核转发给进程，进程有 handler 处理

---

## 五、系统调用 (System Call) 的完整流程

系统调用是用户程序**主动**请求内核服务的唯一合法途径。完整过程：

1. 程序调用库函数（如 `read()`）
2. 库函数校验参数，执行 **`INT` / `SYSCALL` 指令**（软件中断/陷阱）
3. CPU 切换到**内核模式**，保存用户进程状态
4. 根据**陷阱码**查找中断向量，跳转到对应内核处理函数
5. 内核执行服务，切回用户模式
6. 执行 `IRET`，恢复用户进程继续运行

**系统调用的六大类型**：进程控制 (`fork/exit`)、文件管理 (`open/read/write`)、设备操作 (`ioctl`)、信息维护 (`getpid/sleep`)、通信 (`pipe/mmap`)、保护 (`chmod/chown`)。

---

## 考试重点总结

|概念|一句话记忆|
|---|---|
|中断|硬件通知 CPU 发生了事件，CPU 跑去执行 ISR|
|异常 (Fault)|CPU 自己检测到问题（可修复，如缺页）|
|异常 (Abort)|严重到无法恢复，进程终止|
|信号|内核通知进程的软件级中断，有默认 handler 可重载|
|系统调用|用户程序主动陷入内核请求服务的机制|
|SIGKILL (9)|唯一不可屏蔽的进程终止信号|


## 中断
### 为什么需要中断？

想象 CPU 委托打印机打一份文件。如果 CPU 一直盯着打印机等它完成（"忙等待" busy-wait），什么别的事都做不了——这非常浪费。中断解决了这个问题：CPU 把任务交出去，继续干别的，等打印机完成后主动"拍一下 CPU 的肩膀"说"好了！"。

这个 "拍肩膀" 的机制，就是中断。

### 中断控制器（IPC）与两类 IRQ 线

CPU 本身不直接与每个外设"握手"——它们之间有一个专用的硬件中间人：**IPC（Interrupt Programmable Controller，可编程中断控制器）**。

IPC 的三大职责是：将中断路由到正确的 ISR；在 CPU 处理关键代码时推迟低优先级中断；以及支持中断优先级排队。

IRQ 线分为两类：**可屏蔽（Maskable）**的，如键盘、鼠标等普通外设，操作系统可以暂时屏蔽它们；以及**不可屏蔽（NMI，Non-Maskable Interrupt）**的，如内存奇偶校验错误、芯片组故障，这类中断无法被忽略，必须立即处理。用户按下 `Ctrl+Alt+Del` 也走 NMI 路径。

---

### 中断优先级与中断链

中断是可以嵌套的。当 CPU 正在处理一个低优先级的 ISR 时，若来了更高优先级的中断，CPU 会暂停当前 ISR，先去处理高优先级的，回来后再继续。

实际实现中经常使用**中断链（Interrupt Chaining）**：一旦进入某个 ISR，后续同类中断不再触发新的上下文切换，而是在当前 ISR 内部排队处理，避免栈溢出和过度切换开销。
