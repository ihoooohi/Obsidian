# 一、这节课在讲什么？

这一节的核心问题是：

👉 **程序是怎么和外部世界（磁盘、键盘、网络）交互的？**

从第2页可以看到，重点包括：I/O 工作方式、设备类型、接口、OS 提供的服务、buffer/cache

---

# 二、I/O 的基本模型（非常重要）

看第3页：

👉 一个程序其实是这样运行的：

- CPU计算一会（CPU burst）
    
- 发起一次 I/O（比如读文件）
    
- 再计算
    
- 再 I/O ……
    

📌 关键点：

- **I/O通常是最慢的部分**
    
- OS负责管理所有I/O资源
    

👉 所以系统优化的核心之一就是：  
**如何让CPU和I/O不互相浪费时间**

---

# 三、硬件层：I/O是怎么连起来的？

## 1. 总线 + 控制器（第4-6页）

计算机结构大致是：

- CPU
    
- 内存
    
- I/O设备（磁盘、键盘、网卡）
    

它们通过：

- **总线（bus）**连接（比如 PCIe、USB）
    
- **控制器（controller）**负责具体设备操作
    

👉 关键理解：

- CPU **不直接操作设备**
    
- CPU → 控制器 → 设备
    

---

## 2. 控制器寄存器（第7页）

控制器里面有几个重要寄存器：

- command（命令）
    
- data-in / data-out（数据）
    
- status（状态）
    

👉 CPU就是通过读写这些寄存器来控制设备的

---

# 四、CPU 和设备如何交互？

## 1. 最基础方式：轮询（Polling）

第8-9页讲了一个流程：

👉 CPU不断问：  
“设备好了吗？好了没？好了没？”

这叫：  
👉 **busy waiting（忙等待）**

❌ 问题：

- 浪费CPU时间
    
- 效率低
    

---

## 2. 更好的方式：中断（Interrupt）

👉 设备主动通知CPU：

“我好了！”

优点：

- CPU不用傻等
    
- 可以去做别的事
    

---

## 3. 大数据传输：DMA（第11-12页）

问题：

- CPU一点点搬数据太慢
    

解决：  
👉 **DMA（Direct Memory Access）**

- CPU只下命令
    
- DMA控制器直接在：
    
    - 设备 ↔ 内存 之间搬数据
        
- 完成后通知CPU
    

📌 核心思想：  
👉 **“让硬件自己搬数据，CPU别管”**

---

# 五、软件层：操作系统怎么统一管理？

## 1. 抽象（最重要思想之一）

第13页：

👉 设备很多、差异很大怎么办？

答案：  
👉 **抽象（Abstraction）**

- 提供统一接口
    
- 隐藏底层差异
    

---

## 2. 设备驱动（Device Driver）

第14页：

👉 驱动的作用：

- 屏蔽硬件细节
    
- 给OS提供统一接口
    

📌 结论：  
👉 **驱动 = OS 和硬件之间的翻译官**

---

# 六、四种典型I/O接口（考试重点）

## 1. 块设备（Block Device）

如：硬盘

API：

- read / write / seek
    

👉 特点：

- 按“块”读写
    
- 支持随机访问
    

---

## 2. 字符流设备（Character Stream）

如：键盘、鼠标

API：

- get / put（一个字符）
    

👉 特点：

- 一次一个字符
    
- 输入不确定
    

---

## 3. 内存映射（Memory-Mapped I/O）

👉 把设备当成内存用

- 直接读写内存地址
    
- OS负责映射
    

---

## 4. 网络接口（Socket）

👉 网络通信统一成：

- socket()
    
- connect()
    
- select()
    

---

# 七、I/O调用方式（高频考点）

第19页：

## 1. Blocking（阻塞）

- 等I/O完成才继续
    

## 2. Non-blocking（非阻塞）

- 立即返回（可能没数据）
    

## 3. Async（异步）

- 立即返回
    
- 完成后通知你（中断）
    

👉 一句话总结：

- blocking = 等
    
- non-blocking = 不等，但你自己查
    
- async = 不等，系统通知你
    

---

# 八、OS在I/O中的高级优化

第20页总结了一堆服务：

---

## 1. I/O调度（Scheduling）

👉 重新排序请求，提高性能

比如磁盘：

- FCFS
    
- SSTF
    
- SCAN（电梯算法）
    

---

## 2. Buffering（缓冲）

👉 解决速度不匹配

例子：

- 网络慢 → 磁盘快
    

👉 用buffer中转

---

## 3. Caching（缓存）

👉 把常用数据放内存

效果：

- 读更快
    
- 减少磁盘访问
    

---

## 4. Spooling（假脱机）

👉 典型：打印机

- 先写到磁盘队列
    
- 再慢慢打印
    

---

## 5. 错误处理 & 保护

- 重试失败操作
    
- I/O必须通过系统调用（不能乱访问硬件）
    

---

# 九、整节课核心总结（建议背）

你可以记这条主线：

👉 **应用程序 I/O → OS（驱动/抽象）→ 控制器 → 设备**

再加三个关键优化思想：

1. **避免CPU空等**
    
    - interrupt / DMA
        
2. **统一接口**
    
    - abstraction / driver
        
3. **提高性能**
    
    - buffering / caching / scheduling
