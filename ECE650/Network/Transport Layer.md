## TCP(Transmission Control Protocol)

**TCP 的本质：可靠的、面向连接的传输协议**
 **1️⃣ 面向连接（connection-oriented）**

- 通信前要先建立连接（三次握手）
    
- 通信结束要释放连接（四次挥手）
      
 **2️⃣ 端到端的字节流（byte stream）**

- TCP 不关心“消息边界”
    
- 只负责把**一串字节**可靠地从一端送到另一端

### TCP Service Model

- **Full-duplex**：双方可以同时发数据
- **Point-to-point**：只能两个端点（不支持广播/多播）

👉 一个 TCP 连接 = **两个进程之间**

### TCP Protocol

## **一、先记一句“总纲”**

> **TCP：给“字节”编号，用“段”来传，用 ACK 告诉对方我还想要哪个字节**

下面所有图，都是在解释这句话。

---

## **二、字节序列号示意图（最关键）**


假设应用层要发送一串数据：

```
应用层字节流（无限长）：

字节编号 →
1000 1001 1002 1003 ... 1499 1500 1501 ...
|---------------------------------------|
```

👉 **每一个字节都有一个序列号**

---

## **三、TCP 把字节“打包成 segment”**


假设 TCP 决定这样分段：

```
Segment #1
SEQ = 1000
数据长度 = 500 字节

携带的字节：
[1000 .................................. 1499]
```

```
Segment #2
SEQ = 1500
数据长度 = 500 字节

携带的字节：
[1500 .................................. 1999]
```

⚠️ 重点：

- **Sequence Number = 本段第一个字节的编号**
    
- 不是“这是第几个 segment”
    

---

## **四、接收方如何 ACK？（考试高频）**

接收方成功收到 Segment #1 后：

```
已收到字节：
[1000 .................................. 1499]

我接下来想要的是 → 1500
```

于是回一个 ACK：

```
ACK = 1500
```

👉 含义是：


> **1000 ~ 1499 都 OK，请从 1500 开始继续发**

---

## **五、滑动窗口整体示意图（核心机制）**


假设发送窗口大小 = 1000 字节：

```
发送方视角（字节编号轴）：

|---- 已发送并确认 ----|---- 已发送未确认 ----|---- 可发送 ----| 不可发送 |
0                   1000                  2000           3000
                    ↑                     ↑
                  ACK=1000           Window 允许的上限
```

 **发生 ACK 后窗口“滑动”**

收到：

```
ACK = 2000
```

窗口向右滑动：

```
|---- 已确认 ----|---- 已确认 ----|---- 可发送 ----|
0              1000            2000            30000
```

👉 这就是 **sliding window**

---

## **六、ACK + 数据“捎带”（piggyback）**

  

你 PPT 里这句：

  

> May be piggy-backed with data going in the other direction

  

示意图：

```
A → B :  SEQ=1000, data
B → A :  SEQ=5000, ACK=1500, data
```

意思是：

- B 不单独发 ACK
    
- **在回数据的时候顺便把 ACK 填上**
    
- 减少报文数量，提高效率
    

---

## **七、TCP segment 长度为什么有限？**

  

结合你 PPT 的这部分：

```
Segment size limitations:
- Must fit inside IP payload (65535 bytes)
- Limited by MTU (e.g., 1500 bytes for Ethernet)
```

示意：

```
以太网 MTU = 1500

[ Ethernet ]
[   IP    ] 20 bytes
[   TCP   ] 20 bytes
[  DATA   ] ≈ 1460 bytes  ← 这就是 MSS
```

👉 所以：

- TCP 会 **拆分 / 合并应用写入的数据**
    
- 应用层写 1MB
    
- TCP 实际可能发成很多个 segment
    

---

## **八、把整页 PPT 翻译成“人话版”**

  

- > TCP 不按报文编号，而是给**每个字节编号**
    
- > TCP 用 segment 承载字节
    
- > Sequence Number = 本段第一个字节
    
- > ACK = 接收方“下一个想要的字节”
    
- > 滑动窗口控制能发多少、什么时候重传
    
- > segment 大小受 MTU 和 IP 限制
