### fork 会不会立即把整个内存复制一份？

不会，fork 会创建一个子进程，逻辑上复制父进程的地址空间。  
但现代操作系统使用 Copy-On-Write 机制，  
fork 时并不会立即复制整个内存，而是让父子进程共享物理页，并将页标记为只读。  
当某一方发生写操作时，才会触发缺页异常，由内核复制该页。  
这样可以大幅降低 fork 的开销，尤其是在 fork 后紧接 exec 的场景。

### HTTP 和 HTTPS 的区别是什么？

HTTP 是明文传输协议，而 HTTPS 是在 HTTP 基础上加入了 TLS/SSL 加密层。

HTTPS 通过数字证书验证服务器身份，并对数据进行加密传输，防止数据被窃听或篡改。

两者默认端口分别是 80 和 443。

HTTPS 的安全性更高，是目前 Web 的主流协议。

### HTTPS 对称加密和非对称加密如何应用？

> HTTPS 使用了对称加密和非对称加密的组合方式。

> 在 TLS 握手阶段，客户端通过服务器证书获取公钥，使用非对称加密安全地协商会话密钥。

> 当会话密钥建立后，后续的数据传输就使用对称加密算法进行加密，因为对称加密效率更高，适合大量数据传输。

> 因此 HTTPS 的核心思想是：**非对称加密用于密钥交换，对称加密用于数据传输。**

### **消息队列消息发送方式有哪些？**

> 常见的消息发送方式有三种：同步发送、异步发送和单向发送。

> 同步发送会等待 Broker 返回发送结果，可靠性最高，适合关键业务场景；

> 异步发送通过回调获取发送结果，不会阻塞线程，适合高并发场景；

> 单向发送只负责发送消息，不关心结果，性能最好但可靠性最低。

> 此外，一些消息队列还支持事务消息、顺序消息和延迟消息，用于满足不同业务需求。
