# Obsidian 个人博客发布系统 - 需求文档

## 1. 项目概述
利用 Obsidian 作为内容创作端，通过 GitHub 进行版本管理与自动化部署，将笔记发布至火山云服务器，构建一个响应式、高性能且易于扩展的个人博客网站。

## 2. 核心功能需求 (MVP 阶段)
- **笔记同步与转换**：
    - 支持从 Obsidian 文件夹自动提取 Markdown 文件。
    - 处理 Obsidian 特有语法（如 `[[Internal Links]]`、`![[Images]]`）。
    - 支持 Frontmatter 解析（标题、日期、标签、分类）。
- **前端展示**：
    - 响应式设计（适配手机和电脑）。
    - 列表页：按时间或标签展示文章。
    - 详情页：高质量的 Markdown 渲染（支持代码高亮、数学公式）。
    - 搜索功能：支持全文搜索或标题搜索。
- **自动化部署 (CI/CD)**：
    - 当 GitHub 仓库有更新时，自动触发构建并部署到火山云服务器。

## 3. 建议技术栈
- **框架**：Next.js (React) - 优秀的 SEO 支持和高性能静态生成 (SSG)。
- **样式**：Tailwind CSS - 快速开发美观且响应式的 UI。
- **内容处理**：Contentlayer 或 MDX - 将 Markdown 转换为结构化数据。
- **部署**：GitHub Actions + Docker (或直接部署至 Nginx)。
- **存储**：Git 作为主存储，后期 AI 功能可引入向量数据库。

## 4. 系统架构
1. **创作端**：Obsidian (Local) -> Git Push
2. **托管端**：GitHub Repository
3. **构建端**：GitHub Actions (生成静态文件)
4. **运行端**：火山云服务器 (Nginx 托管)

## 5. 可扩展性设计 (未来规划)
- **AI 增强**：
    - **智能搜索**：基于向量检索 (RAG) 实现语义搜索，而不仅仅是关键词匹配。
    - **自动摘要**：使用大模型自动生成文章摘要。
    - **对话式笔记**：用户可以通过聊天的方式询问笔记中的内容。
- **评论系统**：集成 Giscus 或 Valine。
- **多端支持**：增加移动端预览。
