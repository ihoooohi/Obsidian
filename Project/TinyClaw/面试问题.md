## 为什么用 go？

事情是这样的，我在我的 2c2g的服务器上安装了 openclaw 但我发现占的内存太大了，500mb 吃不消，OpenClaw 的官方文档里常见的问题 FAQ 中说了 Node js gateway的大小在500mb 左右
，于是我打算用 go 来重构一个，最后他运行的 gateway 只要 32mb 大大减小，然后我研究了一下发现，TypeScript 本质运行在 Node.js 上，而 Node.js 依赖 V8 VM。 启动时会加载 VM、很多运行时模块，所以即使空载也会有比较高的内存基线。Go 是编译成原生二进制运行，没有 VM，也没有大量运行时模块，所以空载内存会很低。

go 编写的优势在于省内存，可移植（ARM，X86 架构）sftp 到服务器直接跑，启动快

可移植性这一块，直接去我 github 上的 release 里下载二进制文件，然后 sftp 到服务器就能直接跑了，不需要安装 node.js

其实我觉得 go 编写的省内存的优势最大的优势还没在我低配服务器上体现出来，而是在硬件上，比如树莓派，老旧的安卓手机（ARM），我最近关注到一个 github 项目叫 picoclaw 就是，所以后续我会去 picoclaw 上提交 pr，而不是继续我的开发，或者将一些我的开发提交pr 到 picoclaw 上

## 为什么用bus

bus 的设计很符合 go 的并发哲学 ==*"不要通过共享内存来通信，要通过通信来共享内存"*==
