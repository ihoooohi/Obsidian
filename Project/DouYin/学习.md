| Social | 关注、取关 | 抖音关注 |

### 第二阶段：理解技术架构（2-3天）

**面试必问：画一下项目架构图**

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   用户端     │────▶│   Gin API   │────▶│   MySQL    │
└─────────────┘     │  (HTTP服务)  │     └─────────────┘
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
         ┌────────┐  ┌────────┐   ┌────────┐
         │ Redis  │  │RabbitMQ│   │ Worker │
         │(缓存)   │  │(消息队列)│  │(异步消费)│
         └────────┘  └────────┘   └────────┘
```

**数据流向（面试重点）**：

1. **点赞场景**：
   - 用户点赞 → API接收到请求 → **先发MQ消息** → 立即返回给用户
   - Worker异步消费消息 → 写入MySQL + 更新Redis热度

2. **为什么这么做？**
   - 用户体验：点赞响应快（不用等数据库写入）
   - 高并发：削峰填谷，保护数据库

### 第三阶段：死磕8大亮点（面试核心！）

这是面试官最看重的地方，必须能详细说出来：

#### 1. 鉴权缓存自愈机制
```
面试话术：
"我在项目中使用Redis存储JWT Token，但做了一个自愈机制：
1. 鉴权时优先查Redis
2. 如果Redis没有/失败了，回退到MySQL查
3. 查到后再回填到Redis

这样既保证高性能（Redis），又保证高可用（Redis挂了还能用）"
```

#### 2. 分布式锁防击穿
```
面试话术：
"热点数据（比如视频详情）过期瞬间，大量请求同时打到数据库，
这叫'缓存击穿'。我用Redis的SETNX做互斥锁：
- 第一个请求拿到锁，去查数据库
- 其他请求等待或返回兜底数据
- 这样只有一个请求去查数据库"
```

#### 3. 滑动窗口热榜
```
面试话术：
"热榜按分钟分桶存储在Redis ZSET（hot:video:1m:202601011200），
查询时用ZUNIONSTORE合并最近60个桶，生成快照再分页。
好处：减少单个Key的写竞争，保证分页稳定性"
```

#### 4. MQ异常降级直写
```
面试话术：
"如果RabbitMQ不可用怎么办？我做了降级处理：
- 尝试发MQ消息
- 如果发送失败，直接写MySQL/Redis
这样保证'请求成功=数据落地'，不会出现数据丢失"
```

#### 5. 复合游标分页
```
面试话术：
"点赞数排序时，可能有100个视频都是10个赞，
单纯按点赞数分页会重复。我用了'点赞数+ID'复合游标：
WHERE likes_count < ? OR (likes_count = ? AND id < ?)
保证不重复、不遗漏"
```

#### 6. 软硬鉴权兼容
```
面试话术：
"Feed流接口做了两种鉴权模式：
- JWTAuth：强制登录（如关注列表）
- SoftJWTAuth：可选登录（如推荐流）
这样匿名用户也能刷视频，登录后还能显示点赞状态"
```

#### 7. 主动失效一致性
```
面试话术：
"数据变更时（点赞/删除视频），我主动DEL相关缓存：
- 删视频 → 删除视频详情缓存 + Feed缓存
- 点赞 → 删除视频详情缓存（让下次查询最新点赞数）
避免用户看到过期数据"
```

#### 8. Docker一键部署
```
面试话术：
"写了Docker Compose，一行命令启动所有依赖：
docker-compose up -d rabbitmq
便于本地开发联调和生产部署"
```

### 第五阶段：模拟面试问题

**高频面试题及答案要点**：

| 问题 | 回答要点 |
|------|----------|
| 项目遇到的最大挑战？ | 高并发热点数据，用Redis SETNX互斥锁解决击穿问题 |
| 为什么用RabbitMQ？ | 异步解耦、削峰填谷，提高接口响应速度 |
| Redis和MySQL如何选？ | 热点数据放Redis（Token、Feed缓存），持久化放MySQL |
| 如何保证数据一致性？ | 主动失效缓存 + MQ补偿机制 |
| 分页如何保证不重复？ | 复合游标（排序字段+主键） |
