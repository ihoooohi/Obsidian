# Linux 文件系统

## 三层结构总览

Linux 文件系统可以分三层来理解，从外到内依次是：

|层次|是什么|关键概念|
|---|---|---|
|第一层|你看到的样子|目录树、挂载点|
|第二层|管理文件的规则|VFS、ext4、XFS、Btrfs|
|第三层|最底层的存储方式|inode、数据块|

---

## 第一层：目录树

Linux 整个文件系统是一棵统一的树，根节点是 `/`。所有存储介质——本地磁盘、U 盘、网络存储——挂载后都变成树上的一个目录节点。

![[Pasted image 20260406104422.png]]


### 常见目录

|目录|用途|
|---|---|
|`/etc`|系统配置文件（如 `/etc/hosts`、`/etc/fstab`）|
|`/home`|用户主目录（`/home/alice`、`/home/bob`）|
|`/bin`|基础命令（`ls`、`cp`、`mv` 等）|
|`/var`|运行时数据，会持续增长（日志在 `/var/log`）|
|`/dev`|设备文件，体现"一切皆文件"的设计哲学|
|`/tmp`|临时文件，重启后清空（通常是 tmpfs）|
|`/proc`|内核和进程信息的虚拟文件系统|

### 挂载机制

```bash
# 查看当前挂载情况
mount

# 手动挂载 U 盘到 /mnt/usb
mount /dev/sdb1 /mnt/usb

# 开机自动挂载配置文件
cat /etc/fstab
```

## 第二层：VFS 与文件系统类型

![[Pasted image 20260406104454.png]]

### 为什么需要 VFS？

VFS（Virtual File System，虚拟文件系统）是内核中的**统一接口层**，解决的是"解耦"问题。

> [!note] 没有 VFS 会怎样？ 应用程序要读文件，必须知道底层是 ext4 还是 XFS，用不同的接口——新加一种文件系统，所有程序都得改。U 盘插上，`vim` 根本不认识它。

**有了 VFS**：

- 所有程序只跟 VFS 说话，使用统一的 `open() / read() / write()` 系统调用
- 新增文件系统只需实现 VFS 规定的接口，应用程序完全不用改
- `/proc`、网络文件系统（NFS）也能统一管理

```
应用程序（vim、nginx、你的程序）
        ↓  open() / read() / write()
       VFS（虚拟文件系统）
  ↙    ↓    ↘    ↓    ↘
ext4  XFS  Btrfs tmpfs  NFS
        ↓
   磁盘 / SSD / 网络
```


> [!tip] 怎么选？ 普通用途选 **ext4**，稳定成熟，资料最多。XFS 和 Btrfs 有特定需求时再考虑。

## 第三层：inode 与数据块

![[Pasted image 20260406104519.png]]

每个文件在磁盘上被分为两部分：

### inode（元数据）

inode 记录文件"是什么"，但**不包含文件名**：

- 文件类型（普通文件、目录、符号链接…）
- 文件大小
- 权限（`rw-r--r--`）
- 所有者、所属组
- 时间戳（创建、修改、访问）
- 数据块指针（文件内容在哪）
- 链接计数（有多少个文件名指向它）

### 数据块（内容）

inode 指向的磁盘块，存储文件的真正内容。大文件会跨多个数据块。

### 文件名是什么？

**文件名只是目录里的一个标签**，指向某个 inode 号。

```
目录项:
  "hello.txt"  →  inode #1042
  "mylink"     →  inode #1042   ← 硬链接，同一份数据
  "photo.png"  →  inode #2891
```

> [!info] 硬链接的原理 两个不同的文件名可以指向同一个 inode，这就是**硬链接**。删除其中一个文件名，只是减少链接计数，只有计数降为 0 时数据才真正被删除。

```bash
# 查看文件的 inode 号
ls -i hello.txt

# 查看 inode 详细信息
stat hello.txt

# 创建硬链接
ln hello.txt mylink

# 查看 inode 使用情况（inode 耗尽会导致无法创建新文件）
df -i
```
